# Sand Together — 游戏设计

> 本文件定义产品的**玩法逻辑、增长机制和世界观**。
> 视觉规范见 `frontend.md`，系统架构见 `architecture.md`。

---

## 产品定位

结合 Cookie Clicker 的挂机成就感与 Bongo Cat 的解压气质。核心体验：**你的每一次击键，都在为这个瓶子添沙**——沙粒永久沉积，不会消失，劳动的痕迹会一直在那里。这是一个长期运行、持续累积的东西：你今天的工作、明天的工作，都会留下来。

- **形态**：透明置顶桌面叠加层（始终可见，不抢焦点）
- **基调**：诗意的、克制的、轻松的
- **定位**：陪伴型挂机工具 × 数据叙事装置——和朋友各自工作，共享一瓶慢慢涨满的沙

## 核心玩法

### 沙粒系统

- 支持**多玩家**同时参与，每人对应专属颜色沙粒
- 玩家打字/点击 → 对应颜色沙粒从出口**细水流**落下 → 受重力堆积（出沙换算目前是简单的 1 键 1 粒、限速细流；"流量/水龙头"玩法待重设计，见 `backend.md`）
- 沙粒物理：元胞自动机（Falling Sand 算法），**在服务端权威计算**；服务端持有唯一网格并广播给所有客户端，同房间画面完全一致、整张网格持久化（见 `architecture.md`、`backend.md`）

### 房间与玩家

- **进入方式**：玩家默认会创建一个属于自己的新房间并开始记录 progression；也可以通过邀请或输入已有房间号，直接加入别人创建好的房间
- **房间容量**：最多 4 名玩家，地位完全对等，没有"房主"或"访客"之分
- **房间生命周期**：房间数据持久化保存，除非房间内所有人都退出/弃用，否则房间及其数据永久保留
- **玩家数据**：当前只记录 `playerID`（持久身份标识）和 `color`（沙粒颜色，目前默认分配，未来可支持手动选择）。技能、属性等差异化设定属于"游戏化系统"的范畴，当前阶段不实现，所有玩家在这之前都是无差别的

### 沉积机制

- **当前实现**：服务端持有的活动网格就是存档的活跃部分——沙一旦落定不再移动，即成为房间的永久记录；刷新 / 新玩家加入都看到同一份。
- **Stage 3 视觉压缩与展开（已实现）**：活动网格固定大小，深层老沙由**服务端**压缩成薄层（`band`，每条记每列主色 + 行数 + 沙粒数）归档，瓶子因此能无限往上堆而不靠把网格无限加大。压缩是**后台存储手段**、对用户透明：归档带在客户端按**真实高度展开**渲染，向下滚动到那段即自动展开成连续历史地层。注意压缩**有损**（每列只留主色），"展开"是把主色拉回真实高度的彩色层、非逐像素还原。机制见 `stage3-compression.md` / `backend.md`。
- **二级压缩（暂不做）**：归档无限增长，超深历史可再加一层压缩。

### 成就系统

当玩家的累计进度达到某个里程碑时，生成一个可见的"成就物件"落入沙中，随后被沙粒永久掩埋，封存于历史层。

成就物件是进度的具象化表达，表现形式可以多样，例如：
- **像素文物**：菊石 Ammonite、软盘 Floppy、灯泡 Bulb 等像素 sprite，作为刚性像素落入沙中
- 未来也可以是其他形态（徽章、文字、自定义图案等）

核心逻辑：物件落入、被埋、消失在沙层深处——成就不是用来炫耀的，而是被时间压住的。

## 游戏化系统（道具 & Build）

> ⚠️ 本模块为**临时设计样例**，当前阶段不优先实现。具体机制待后续迭代确认。

核心思路：玩家通过长期累积的 progress 获得某种"资源"，用来解锁或购买道具，从而丰富"往瓶子里填沙"的方式。目标体验是挂机类游戏的 build 感——你的积累可以反过来改变你积累的方式。

所有子系统都是围绕这一目的的游戏化手段：

### 赌注系统（样例）

- 玩家可快速勾选"1分钟对赌"：预测未来1分钟内自己的击键数区间
- 命中预测 → 基础掉落率提升（细水管 → 粗水管）
- 可与社交连锁机制联动（合作赌注）

### 暴击机制（样例）

- 打字时有概率触发"暴击"：单次击键掉落大量闪光沙粒
- 清脆音效 + 视觉特效，突然发生
- 可通过道具提升暴击概率或效果

### 非操作性累积（样例）

- 无需主动操作的持续累积，不只是"离线收益"，更是一种背景存在感
- 无人打字时，系统仍会缓慢生成沙粒，瓶子始终是活着的

## 边界说明

| 问题 | 决策 |
|---|---|
| 输入监听范围 | 全局系统键盘（不限定特定应用），只计频率不记内容 |
| 物理性能预算 | 活跃粒子上限 5000 颗，超出自动压缩/融合底部 |
| 鼠标穿透 | 点击沙粒区域穿透到桌面（始终置顶但不拦截操作） |
| 网络同步 | **服务端权威**：物理在服务端跑，客户端只发输入（ticks）、收网格增量、纯渲染。同房间所有人画面**逐帧完全一致**，整张网格由服务端持久化；新玩家加入直接收完整快照，不重放历史 |
| 房间容量 | 最多 4 人，地位完全对等，出口位置居中对称排列（详见 `frontend.md`），位置只影响新沙粒从哪开始落下，不影响已冻结的历史像素 |
| 终局机制 | 无终局，沉积层无限累积 |
| 防作弊 | 原型阶段客户端自报，后期视需要改服务端权威校验 |
